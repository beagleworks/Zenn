## 概要

Claude Code on the web(以下CC on the web)とClaude Code CLIには、タスクを相互移動できる機能があります。
つまり、CC on the webで行っている作業をローカルに持ち込んで検証したり、ローカルの作業をCC on the web上に投げることができます。
本記事では、その具体的な方法について主に初心者向けの解説を行っていきます。

## CC on the webからローカルへのタスク移動(セッション移動)

WebからCLIへは、特定のタスクというよりはセッション自体の移動に近いイメージです。
方法については、実は私の過去の記事「Claude Code on the webを触ってみた」でも少し触れています。

https://zenn.dev/beagle/articles/bc6ef88dd68615#%E3%80%8Ccli%E3%81%A7%E9%96%8B%E3%81%8F%E3%80%8D--%E3%82%BB%E3%83%83%E3%82%B7%E3%83%A7%E3%83%B3%E3%81%AE%E3%83%AD%E3%83%BC%E3%82%AB%E3%83%AB%E3%81%B8%E3%81%AE%E7%A7%BB%E8%A1%8C

> まずはGitHub上にある現在の作業ブランチをローカルに引っ張ってきます。
> そしてこの「CLIで開く」ボタンをクリックすると、クリップボードにclaudeコマンドがコピーされます。
> あとは、ローカルのターミナルで作業ブランチに移動し、コマンドをペーストして実行すればこのセッションをローカルで再開できます。

この記事を書いたときに、複数の方から「GitやGitHubの操作に慣れていないので具体的な手順がわからない」という声を頂きました。
ですので、ここでは具体的な手順を示します。

### 1. 使っているGitHubリポジトリをローカルにクローン

ターミナルで 
`git clone <リポジトリURL>`
と入力します。
GitHub CLIが使える場合は
`gh repo clone <ユーザー名/リポジトリ名>`
でもOKです。

クローンしたら
`cd <リポジトリ名>`
でクローン先のディレクトリに移動します。

### 2. 作業ブランチに移動

CC on the webでの作業は、mainブランチではなく、新たな作業ブランチを作成して行われています。
作業ブランチ名は、CC on the webの右下部チャット欄の上に表示されています。
一般的には `claude/<何かしらの文字列>` という形式になっているはずです。

下の図のように、PRを作成ボタンの左側にブランチ名が表示されており、欄の右端にブランチ名をコピーできるボタンがあります。
![](/images/img0016.png)

ブランチ名をコピーして
`git switch <ブランチ名>`

と入力します。
これでローカルにも同名のブランチが作成され、そのブランチに移動した状態になります。

### 3. CC on the webで「CLIで開く」ボタンをクリック

`claude --teleport <セッションID>`
というコマンドがクリップボードにコピーされますので、貼り付けて実行します。
するとClaude Codeが起動してCC on the webでのセッションが復元されます。

これでローカルのClaude Codeで作業を再開できます。

## 補足1 Git Worktreeを利用する方法

例えばこのような状況を考えてみます。

「はじめからCC on the web上で作業を進め、作業結果を何度かmainブランチにマージしたGitHubリポジトリがある。今、ある作業ブランチをローカルで検証したいが、同様に現状のmainブランチもローカルで検証を行いたい。」

このような場合は、**Git Worktree**を利用するのがおすすめです。

Git Worktreeは、1つのリポジトリにある複数のブランチを、それぞれ別々のフォルダで展開できる機能です。
Gitの標準機能として提供されていますので、Gitがインストールされていればすぐに利用可能です。

![](/images/img0018.png =400x)

リポジトリをクローンしたあと
`git worktree add ../<任意のディレクトリ名> <ブランチ名>`
とします。

これで、クローン先のディレクトリはmainブランチ、Worktreeで指定したディレクトリはリモート作業ブランチとなり、ファイルの状態がそれぞれのブランチのものになります。
つまり、これで各ブランチを個別に検証できます。
また、この場合はWorktree側ディレクトリでCC on the webのセッションを復元できます。

なお、Git Worktreeについての詳しい使い方はこちらの記事が参考になります。

https://zenn.dev/hiraoku/articles/56f4f9ffc6d186

## 補足2 Git Worktree Runnerを利用する方法

「Git Worktreeはなんとなくわかるが、ディレクトリ移動等の操作が煩雑に感じる」といった場合では**Git Worktree Runner**を利用する方法もあります。

https://github.com/coderabbitai/git-worktree-runner

これはAIレビューツールとして有名な**CodeRabbit**が提供しているツールで、Git Worktreeをより簡単に扱うことができるようになります。

https://www.coderabbit.ai/

インストールや使い方については上記GitHubリポジトリのREADMEを参照してください。
今、Git Worktree Runnerのインストールが終了したものとし、補足1と同じ状況を考えてみます。

リポジトリをクローンしたあと
`git gtr new <リモートブランチ名> --name <ローカルブランチ名>`
とします。ブランチ名が長いので「ローカルブランチ名」のところに任意の名前をつけておくとよいでしょう。
これでリモートブランチがローカルリポジトリとして展開されました。

この新たなブランチを実際に開くときには
`git gtr editor <ローカルブランチ名> --editor <IDE名>`
とすると、このブランチをカレントディレクトリとして新たなIDEのウィンドウが開きます。
対応IDEはvscode, cursor, zedのようです。

また、ターミナルを利用している場合は
`cd "$(git gtr go <ローカルブランチ名>)"`
でそのブランチのディレクトリに移動できます。

以上のように、Worktreeのディレクトリがどこにあるのかを意識する必要がなくなるため、非常に便利なツールです。

## Claude Code(CLI)からCC on the webへのタスク移動方法

こちらは非常に簡単ですが、実装されてから日が浅いため(11月中旬実装)、あまり知られていないように思えます。
OikonさんのXのポストが参考になります。
https://twitter.com/oikon48/status/1990930519731941749

手順はClaude CodeのChat欄に`&`を入力し、続けて作業内容を入力して送信するだけです。
ただし、**作業ディレクトリがGit管理されており、リモートリポジトリがGitHubに存在している状態**になっている必要があります。

具体的に見ていきましょう。例えば
`& feature.tsが正しく動作しない問題を修正してください。`
とします。しばらく待つとCC on the webで該当の作業が始まります。
このとき、同時にCC on the webへのURLリンクが表示されますので、ブラウザで開くと進行状況を確認できます。
または`/tasks`コマンドを使えばClaude Code内で進捗が確認できます。

## まとめ

最近のアップデートにより、CC on the webとClaude Code(CLI)の連携が強化され、タスク/セッションの移動が簡単になりました。

CC on the webからClaude Codeへのセッション移動は、生成コードのローカル検証だけでなく、CC on the webで作成したコードをローカル環境で本格的に開発したい場合にも便利です。

一方、Claude CodeからCC on the webへのタスク移動については、活用シーンがイメージしにくいかもしれません。
例えば以下のようなケースで有効です

- 外出前にCC on the webへタスクを投げておき、移動中にスマホで進行状況を確認する
- 時間のかかる作業をCC on the webに任せ、ローカルのClaude Codeでは別の作業を進める
- 該当の作業を頭のコンテキストにおいておきたくないので、CC on the webに投げて頭を整理する

開発に必須の機能というわけではありませんが、知っておくと便利な場面はありそうだと思っております。
